# CS-300 Journal

## What was the problem you were solving in the projects for this course?
I was building an application for ABCU’s computer science advisors to access student course information. The goal was to use algorithms and data structures to quickly and efficiently navigate a course list with prerequisites.  

## How did you approach the problem? Consider why data structures are important to understand.
I approached the project from a user perspective. Technology helps automate tasks, saving time and increasing accuracy and efficiency. Even though my project only included a few courses, I had to consider scalability. In a real-world scenario, there are many courses, and requirements may change over time. Choosing the right data structures helps store and retrieve data efficiently and improves performance.  

## How did you overcome any roadblocks you encountered while going through the activities or project?
I relied on logic and online research to work through problems. Sometimes I felt imposter syndrome because I was building applications without fully understanding what was happening under the hood. Focusing on logic helped me solve problems and discover new ways to approach challenges.  

## How has your work on this project expanded your approach to designing software and developing programs?
The project taught me to think more critically about data and how programs use it. Software design isn’t just about visuals—it’s also about memory usage, performance, and efficiency.  

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
This project made me rethink how I structure my code. I now pay more attention to internal design, not just the visual layout. For example, I’m building a catalog of my collector’s items, and the main challenge is deciding how to organize the database and navigate the data efficiently.
